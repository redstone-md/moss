why moss

⛁

Serverless discovery

Peers rendezvous through public BitTorrent trackers and the DHT — and in the browser, through WebRTC signaling. No bootstrap servers to run or trust.

🔒

Encrypted by default

Every session is a Noise XX handshake with identity binding and ChaCha20-Poly1305. UDP is obfuscated to resist DPI fingerprinting.

◉

Trustless telemetry

A gossiped, hash-chained CRDT lets anyone see node count and bandwidth — privacy-preserving, reproducible, and verified in your browser.

⟁

NAT traversal

Autonomous hole-punching with supernode relay fallback reaches peers across full-cone, symmetric, and carrier-grade NAT.

⚙

Embeddable core

A small C ABI exports the runtime to C, C++, C#, Python, and Rust — or compile it to WebAssembly and run a peer in a tab.

❖

Browser-native

mosh-web runs the full Moss peer in WebAssembly over WebRTC — encrypted pub/sub between tabs and devices, no install.

how it works

01 · discover

find peers

A node derives a private infohash from the mesh id + PSK and announces to trackers, the DHT, or a WebRTC signaling room.

02 · connect

encrypt & route

Sessions complete a Noise handshake, then GossipSub-style pub/sub routes messages with peer scoring and relay fallback.

03 · observe

verify the network

Nodes gossip a privacy-preserving telemetry CRDT. The explorer recomputes and verifies the hash chain locally — no trusted source.
